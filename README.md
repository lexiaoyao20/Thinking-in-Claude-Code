# Thinking in Claude Code

> 从本仓库整理出的重点不是“怎么用一个工具”，而是“怎样从一套大型 AI Agent 源码里提炼出可迁移的设计经验”。

本仓库的主体内容在 [`book/`](./book/)。

如果你只想直接开始阅读，请先看：

- [《Claude Code 源码编程思想》导读](./book/README.md)
- [前言：为什么要读一个 CLI 工具的源码](./book/00-preface.md)
- [附录 A：源码导航速查](./book/appendix-a.md)

---

## 这是什么

这是一个围绕 Claude Code 源码展开的书稿型仓库。它不是用户手册，也不是逐文件注释，而是把一套大型 AI Agent 系统拆开来看，回答这些更重要的问题：

- 为什么它要这样分层？
- 为什么 Agent 不是“一次调用”，而是一个循环？
- 为什么权限、上下文、并发、扩展这些问题，最终都会变成架构问题？
- 哪些做法可以从 Claude Code 迁移到你自己的项目里？

换句话说，这个仓库最值得看的部分不是零散代码片段，而是 `book/` 目录里已经整理好的整套讲解。

## 仓库重点

### `book/`

这是仓库的主角，包含整本书的正文、导读、附录和术语表。

- 共 17 章正文，外加前言、附录、术语表
- 当前总量约 1.3 万行
- 组织方式已经从“单文件散放”整理为 `book/` 目录集中管理

如果你是第一次来这个仓库，建议把这里当成默认入口。

### `src/`

这是书中分析所对应的源码材料。阅读时它更像“对照区”而不是“起点”。

适合两种场景：

- 你读到某章时，想回头看对应实现
- 你准备把书里的思路迁移到自己的项目里，想确认原始设计细节

### `optimized_prompt.md`

这是额外的研究材料，不是这本书的主阅读路径。

## 这本书适合谁

- 正在做 AI Agent、代码助手、自动化工作流的人
- 想看一套“已经长大”的终端产品是怎么组织起来的人
- 想研究大型 TypeScript 项目如何处理边界、约束、恢复能力的人
- 不满足于“会用”，更关心“为什么这样设计”的人

如果你只是想知道某个命令怎么用，这个仓库不是最快入口。

---

## 阅读时怎么和源码配合

一个比较顺手的方式是：

1. 先在 `book/` 里读某一章，先理解问题和设计取舍
2. 再打开对应的 `src/` 文件，看书里提到的那几个关键位置
3. 如果想快速定位，不要盲翻，直接用 [附录 A](./book/appendix-a.md)

这样阅读的效率会比直接从 `src/` 硬啃高很多。
